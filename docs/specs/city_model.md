# Модуль `city_model`

## Назначение

Модуль представляет декларативную модель города, проверяет её инварианты и
собирает файловые артефакты для других компонентов CyberCity. Он не исполняет
сценарии и не управляет инфраструктурой.

## Поведение

- `check` читает модель и сообщает все нарушения без создания артефактов.
- `build` после успешной проверки воспроизводимо создаёт набор артефактов.
- `init` создаёт исходную структуру новой организации.

## Контракты

Вход — YAML из `organizations/`; выход — документированные в архитектуре файлы
`engine.zip`, `topology.json` и сопутствующие представления. Событие
`city.build.completed` остаётся запланированным и не выдается за реализованное.

## Инварианты и данные

Некорректная модель не попадает в сборку. При заданном `--seed` адресация
воспроизводима. Исходные YAML являются владельцем декларативных данных.

## Ошибки и зависимости

Ошибки чтения, схемы, межполевых правил и записи артефактов возвращаются CLI с
ненулевым кодом. Внешние потребители не вызываются синхронно.

## Проверка

| Поведение | Тест или пробная проверка |
|---|---|
| Проверка модели | `uv run pytest -q tests/domain tests/data` |
| Команды CLI | `uv run pytest -q tests/controllers` |
| Сборка пакета | `uv build` |
