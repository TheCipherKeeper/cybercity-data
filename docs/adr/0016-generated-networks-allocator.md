# ADR-0016: Сети и IP генерируются `allocator.py`; декларативная модель описывает только топологию

## Status

Accepted

## Context

Декларирование IP/CIDR вручную (ADR-0003, v2.0) размывало модель адресной
бухгалтерией: структура важнее адресов, а адреса должны быть детерминированными
и воспроизводимыми для replay и тестов.

## Decision

Сети и IP-адреса генерируются `allocator.py` из топологии:

- `org_index` — второй октет городского пространства `10.0.0.0/8`;
- `net_cidr` — CIDR на сеть по её `kind` (`dmz`/`lan`/`ot`/`mgmt`/`internet`);
- `svc_ip` — `bind_ip` сервиса внутри его сети, начиная с `.10`.

Декларативная модель описывает только топологию: организации, роли сетей,
размещение сервисов, связи. Воспроизводимо через `--seed`; без флага — свежая
случайная аллокация на каждой сборке.

## Consequences

- Модель сфокусирована на структуре, а не на адресах.
- IP/CIDR детерминированы и воспроизводимы.
- ADR-0003 помечен `Superseded`. Это решение v3.0.
